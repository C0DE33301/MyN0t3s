## The `find` command
### Find multiple files
`find / -type f \( -name 8V2L -o -name bny0 -o -name c4ZX -o -name D8B3 -o -name FHl1 -o -name oiMO -o -name PFbD -o -name rmfX -o -name SRSq -o -name uqyw -o -name v2Vb -o -name X1Uy  \) -exec BASE-COMMAND-HERE {} \; 2>>/dev/null`
- `find /`: Start from the root directory.
- `-type f`: Look for files only (not directories).
- `\()` & `\)`: Used to group conditions.
- `-name 8V2L -o -name bny0 -o -name c4ZX -o ...`: Will match files with any of the names listed
    - `-name`: File name.
    - `-o`: **OR**
- `-exec BASE-COMMAND-HERE {} \;`: To execute a command on each file that matches the search.
    - `{}`: A placeholder for the matched file names.
    - `\;`: The end of the command to be executed.
- `2>>/dev/null`: Redirects error messages to `/dev/null`, discarding any error output. To keep the output clean by ignoring `permission denied errors` for example.

## Log files
### The `wtmp` file
A historical record of all login and logout activities for users on the system. View its contents using the last command, which displays a list of past logins. `last -f /var/log/wtmp`
